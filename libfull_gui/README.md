# libful GUI

Проста веб-сторінка для роботи бібліотекаря й адміністратора з `libful API`.

GUI не потребує `npm`, `node_modules` або збірки. У директорії є статичні файли:

- `index.html` — структура сторінки;
- `styles.css` — оформлення;
- `app.js` — запити до API та поведінка інтерфейсу;
- `server.py` — локальний статичний сервер і proxy до FastAPI.

## Запуск

Спочатку запусти бекенд API з кореня репозиторію:

```bash
source .venv/bin/activate
uvicorn libful_api.main:app --reload
```

Потім в окремому терміналі запусти GUI:

```bash
python3 libfull_gui/server.py
```

Відкрий:

```text
http://127.0.0.1:8080
```

За замовчуванням GUI проксіює запити `/api/v1/...` на:

```text
http://127.0.0.1:8000
```

Якщо API працює на іншій адресі:

```bash
python3 libfull_gui/server.py --port 8080 --api http://127.0.0.1:8001
```

Якщо порт `8080` зайнятий:

```bash
python3 libfull_gui/server.py --port 8081
```

Тоді відкрий `http://127.0.0.1:8081`.

## Авторизація

У верхній панелі введи:

- `API`: `/api/v1`
- логін користувача;
- пароль користувача.

API використовує HTTP Basic Auth. Логін зберігається в `localStorage`, пароль лише в `sessionStorage` поточної вкладки браузера.

Якщо в базі ще немає адміністратора, створи першого користувача у вкладці `Читачі`. API дозволяє створити першого користувача без авторизації та автоматично надає йому роль `admin`. Після цього для захищених операцій потрібно входити під користувачем з відповідними правами.

## Розділи

- `Огляд` — короткі лічильники, останні видачі, неоплачені штрафи.
- `Читачі` — створення, пошук, список користувачів і реєстрація візиту.
- `Каталог` — автори, жанри, книги, пошук книг.
- `Примірники` — створення та фільтрація фізичних примірників.
- `Видача` — видача, повернення, зміна дедлайну, історія та прострочення.
- `Штрафи` — перегляд, preview і позначення штрафів як оплачених.
- `Адмін` — додавання та прибирання ролей `admin` / `librarian`.

## Права доступу

Інтерфейс не приховує вкладки за ролями, але API все одно перевіряє права кожного запиту.

- `admin` може керувати користувачами, ролями, каталогом, відвідуваннями, видачами та штрафами.
- `librarian` може працювати з каталогом, відвідуваннями, видачами й штрафами.
- Користувач без ролей не зможе виконувати захищені операції.

Якщо API повертає `403 Forbidden`, це означає, що поточний логін не має потрібного права.

## Типовий робочий сценарій

1. Запусти API на `http://127.0.0.1:8000`.
2. Запусти GUI на `http://127.0.0.1:8080`.
3. У верхній панелі залиш `API` як `/api/v1`.
4. Увійди під `admin` або `librarian`.
5. Додай авторів і жанри у вкладці `Каталог`.
6. Додай книги й фізичні примірники.
7. Зареєструй читачів у вкладці `Читачі`.
8. Видавай і повертай книги у вкладці `Видача`.
9. Перевіряй прострочення та штрафи у вкладках `Видача` і `Штрафи`.

## Діагностика

Якщо сторінка відкривається, але запити не працюють:

- перевір, що FastAPI запущений;
- перевір адресу proxy в консолі `server.py`;
- залиш поле `API` як `/api/v1`, якщо використовуєш `server.py`;
- перевір логін, пароль і роль користувача;
- відкрий `http://127.0.0.1:8000/docs`, щоб переконатися, що API відповідає.

Якщо потрібно звертатися до API напряму без proxy, вкажи повну адресу в полі `API`, наприклад `http://127.0.0.1:8000/api/v1`. Для такого режиму бекенд має дозволяти CORS у браузері.
