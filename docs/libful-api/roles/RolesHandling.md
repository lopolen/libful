# Обробка ролей користувачів

Доступ до API контролюється через HTTP Basic Auth. Логіном є `username`, паролем є значення, яке передали в поле `password` під час створення або оновлення користувача. API зберігає тільки `password_hash`.

## Таблиці

Ролі зберігаються в таблицях:

- `roles` — довідник ролей. Поле `name` є унікальним.
- `users_roles` — many-to-many зв'язок між користувачами й ролями.

Міграція `20260507_0007` створює стандартні ролі:

- `admin`
- `librarian`

Кастомні ролі через API не створюються. Права ролей визначені в коді в `libful_api/core/permissions.py`.

## Bootstrap першого адміністратора

Щоб не заблокувати систему після першого запуску або міграції старої БД, якщо ще немає жодного користувача з роллю `admin`, наступний створений користувач може бути створений без авторизації через `POST /api/v1/users/`. Він автоматично отримує роль `admin`, навіть якщо поле `roles` не передано.

Коли в системі вже є хоча б один `admin`, створення наступних користувачів вимагає право `manage_users`.

Користувач із роллю повинен мати пароль. API не дозволяє створити користувача з роллю без пароля або додати роль користувачу, у якого `password_hash = null`.

## Права ролей

`admin` має всі права:

- `read_users`
- `manage_users`
- `manage_roles`
- `read_catalog`
- `manage_catalog`
- `manage_check_ins`
- `manage_book_rents`
- `manage_fines`

`librarian` має операційні права бібліотекаря:

- `read_users`
- `read_catalog`
- `manage_catalog`
- `manage_check_ins`
- `manage_book_rents`
- `manage_fines`

Користувач без ролей може пройти автентифікацію, але не має доступу до жодної захищеної операції.

## Керування ролями

Ролі користувача керуються через endpoint-и:

- `GET /api/v1/users/{username}/roles` — переглянути ролі користувача.
- `PUT /api/v1/users/{username}/roles/{role_name}` — додати роль користувачу.
- `DELETE /api/v1/users/{username}/roles/{role_name}` — прибрати роль користувача.
- `GET /api/v1/roles/` — переглянути стандартні ролі.

Додавати й прибирати ролі може тільки користувач із правом `manage_roles`, тобто `admin`.

API не дозволяє видалити останнього адміністратора або прибрати в останнього адміністратора роль `admin`; такі операції повертають `409 Conflict`.
