# Обробка ролей користувачів

Зберігання ролей користувачів забезпечують таблиці roles та users_roles.
Таблиця roles зберігає назву ролі та унікальний ідентифікатор (id). 
Таблиця users_roles є many-to-many таблицею, яка простою мовою зберігає які з ролей має користувач.
Відсутність ролі у користувача означає, що він не може виконувати жодної операції

Операції які може виконувати користувач певнох ролі визначає безпосередньо сервіс (код бізнес логіки).
Тому таблиця roles ПОВИНА мати ТІЛЬКИ ці ролі:
    1. admin
    2. librarian

Наразі дизайн БД та API не передбачає кастомізацію ролей, оскільки не існує
таблиці, яка б визначала права кожної з ролей, й логіки яка б перевіряла це все.
Однак саме існування таблиці roles дозволяє значно спростити покращення системи ролів в майбутньому.
