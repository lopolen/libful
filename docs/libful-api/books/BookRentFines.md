# Дедлайни повернення та штрафи за оренду книг

Оренда книги зберігається в таблиці `book_rents`.

Поле `due_at` є дедлайном повернення книги. Якщо `due_at` не заданий, оренда не вважається простроченою.

## Прострочення

Оренда вважається простроченою, якщо:

- книга ще не повернена і поточний час більший за `due_at`
- або книга повернена після `due_at`

Штраф за прострочення наразі захардкоджений:

```text
5 умовних гривень за кожен день прострочки
```

Неповний день прострочки рахується як повний день. Наприклад, якщо книга прострочена на 2 години, це 1 день прострочки.

## Пошкодження

Якщо при поверненні книги `return_status = damaged`, створюється штраф за пошкодження.

Штраф за пошкодження наразі захардкоджений:

```text
100 умовних гривень
```

## Статуси повернення

Для `book_rents.return_status` дозволені значення:

- `available`
- `damaged`
- `lost`

Автоматично створюються такі типи штрафів:

- `overdue` — прострочення дедлайну
- `damage` — пошкодження книги

## Важливо

Суми штрафів зараз є частиною бізнес-логіки в коді, а не окремою таблицею налаштувань. Якщо правила зміняться, потрібно оновити:

- сервіс розрахунку штрафів
- API-документацію
- тести
- цю документацію
- міграції або довідникові таблиці, якщо ставки стануть конфігурованими

## API

Основні endpoint-и:

- `POST /api/v1/book-rents/issue` — видати примірник книги користувачу. `due_at` можна передати одразу як дедлайн повернення.
- `PATCH /api/v1/book-rents/{book_rent_id}/deadline` — змінити дедлайн активної оренди.
- `GET /api/v1/book-rents/overdue` — переглянути активні прострочені оренди.
- `GET /api/v1/book-rents/{book_rent_id}/fines/preview` — попередньо розрахувати можливі штрафи без запису в БД.
- `POST /api/v1/book-rents/{book_rent_id}/return` — повернути книгу. Під час повернення автоматично створюються штрафи за прострочку та пошкодження.
- `GET /api/v1/book-rents/fines` — переглянути створені штрафи.
- `POST /api/v1/book-rents/fines/{fine_id}/pay` — позначити штраф як оплачений.
